# P02/P03 重新设计方案

## 背景

P02（申请理由）和 P03（拟培养方向）需要重新设计内容逻辑和布局。P02 纯描述现状与问题，不含内涵框和团队能力；P03 针对 P02 的不足逐一提出解决方案。

## P02「申请理由」

### 内容逻辑

石窟大背景 → 现有具身解决方案 → 仍存在的不足（纯描述，无团队能力）

### 布局结构

1. **标题栏**：深蓝 `#1E386B` 全宽（高≈78px），白色粗体「申请理由」+ 橙色渐变编号圆 `01`，无内涵框
2. **双卡片并排**（方案 D 风格：上文下图）
   - 左卡片：浅蓝底 `#DFEAFA`，① 石窟文化遗产保护的紧迫性 + 下方配图（yungang_weathered.png）
   - 右卡片：浅绿底 `#EAF1DC`，绿边框 `#97B752`，② 具身智能已有应用（三条进展要点）+ 下方配图（embodied_active_perception.png）
3. **③ 红框整行**（`#C00000` 边框，白底）：三大技术空白写为一行
   - ① 多模态感知与数字孪生建模技术尚处起步 ② 残损造像虚拟复原推演缺乏智能化方法 ③ 物理感知与虚拟推演闭环交互机制尚未实现
4. **底部总结条**：红框白底矩形，红色粗体文字「亟需开展针对性研究攻关」

### 配图

- 石窟风化：`yungang_weathered.png`（白色/透明背景）
- 具身感知：`embodied_active_perception.png`（白色/透明背景）

### 字体规格

- 标题栏：48px（36pt）
- 卡片标题：24px（18pt）
- 卡片正文/要点：21.33px（16pt）
- 不足区文字：21.33px（16pt）
- 底部总结：28px（21pt）
- 最小字号：21.33px（16pt）

## P03「拟培养方向」

### 内容逻辑

针对 P02 提出的三大不足，逐一提出解决方案和研究路线

### 布局结构

1. **标题栏**：深蓝 `#1E386B` 全宽，白色粗体「拟培养方向」+ 橙色渐变编号圆 `02`
2. **内涵框**（保留）：浅绿底 `#EAF1DC`，绿边框 `#97B752`，青绿「内涵」标签
   - 内容：以具身智能为核心路线，感知建模 → 虚拟复原 → 虚实闭环，逐步攻克技术空白
3. **三张卡片并排**，每张结构：
   - **上方浅红区**（`#FFF0F0` 底，`#C00000` 分隔线）：来自 P02 的具体问题描述
   - **下方白色区**：
     - 方案标题（深蓝/青绿交替标题条）
     - 配图
     - 关键要点（2-3条，红色关键词高亮）
4. **底部总结条**：红框白底，「三个方向逐一攻克石窟场景的关键技术空白」

### 三张卡片内容

| 卡片 | 问题（上方） | 方案（下方） |
|------|-------------|-------------|
| ① | 多模态感知与数字孪生建模尚处起步 | 具身多模态感知与孪生建模：多模态传感器 + 六维位姿 + 数字孪生模型 |
| ② | 虚拟复原推演缺乏智能化方法 | 融合先验的虚拟复原推演：考古知识嵌入 + 残片智能匹配 + 李君教授先验支撑 |
| ③ | 虚实融合闭环交互机制尚未实现 | 虚实融合闭环交互：感知→推演→引导再感知 + 主动虚实融合范式转变 |

### 配图

- 感知建模：`p03_v2_perception.png`
- 虚拟复原：`p03_v2_recon.png`
- 闭环交互：`p03_v2_loop.png`

### 字体规格

同 P02，所有正文 ≥21.33px（16pt），底部 28px（21pt）

## 技术约束

- 画布：1280×720，viewBox "0 0 1280 720"
- 字体：`"Microsoft YaHei", Arial, sans-serif`
- SVG px × 0.75 = PPT pt
- 禁用：rgba()、foreignObject、mask、class/style、`<g opacity>`
- 配图均使用白色/透明背景
