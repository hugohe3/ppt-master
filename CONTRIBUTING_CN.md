# 参与 PPT Master 贡献

感谢你有兴趣参与项目贡献！本指南将帮助你快速上手。

## 贡献方式

- **模板** — 新的布局模板或视觉风格
- **图表** — 新增图表类型或 SVG 图表模板
- **图标** — 为图标库贡献矢量图标
- **脚本** — 改进源文件转换脚本（PDF/DOCX → Markdown）或后处理脚本（SVG → PPTX）
- **文档** — 澄清、翻译或编写新的指南
- **Bug 报告** — 可复现的问题，附带清晰的描述
- **想法** — 功能请求和设计建议

## 快速开始

### 前置条件

- **Python 3.10+** — 唯一的必需依赖
- **Node.js 18+** 和 **Pandoc** 属于边缘场景的备用方案，99% 的贡献者用不到；只有在处理需要它们的具体路径时才安装。详见 [README_CN-快速开始](./README_CN.md#1-前置条件) 了解各自的适用场景。

### 环境搭建

```bash
git clone https://github.com/hugohe3/ppt-master.git
cd ppt-master
pip install -r requirements.txt
```

## 提交 PR 前

PPT Master 由个人维护，审核精力有限。为了让大家顺畅地协作：

- **轻微修复**（拼写错误、一行式的使用说明/文档修正、明显但很轻微的不一致）— 请直接提 issue 而不是开 PR。清晰的问题报告通常比 PR 让维护者更快直接处理
- **针对性 Bug 修复** — 如果修复的内容独立完整、有清晰的复现步骤且经过本地验证，欢迎提交 PR
- **大型功能、新后端或新的抽象层** — 请先提 issue 讨论适用性和方向。未经事先讨论的 PR 可能被直接关闭而不做详细审查
- **重构、结构性变更、大面积清理或工作流改动** — 请先提 issue，因为本项目有意保持当前结构。
  
这不是在设门槛——这是在节省你的时间。一个 PR 应该是一项有意义、可独立审查的变更，而不是明明可以通过提 issue 就能更快修补问题的几行代码。与其提交一个500行但方向错误的 PR ，不如先提交一个10行的 issue 说明问题。

## 我们接受 / 不接受什么

**推荐：**
- 有清晰复现步骤的 Bug 修复
- 新的布局模板、图表模板、图标
- 文档澄清和翻译
- 遵循现有 `image_backends/` 模式的额外图片后端
- 约束范围内的 SVG 质量改进

**避免（请不要为此提交 PR）：**
- 引入 `uv`、`poetry` 或其他工具作为必需依赖 — `pip + requirements.txt` 是唯一的官方安装方式
- 添加 CI、测试框架、pre-commit hooks 或 lint 工程配套 — 个人维护项目刻意将这些排除
- 将 skill 重新打包为 CLI、SaaS、桌面应用或安装器 — PPT Master 本质上是供 AI IDE 使用的对话式 skill
- 架构重构或大规模重命名 — 仅接受增量清理
- 与实际修复无关的"顺手"格式美化

如果你不确定，提 issue 询问——任何时候都欢迎。

## 贡献流程

1. **Fork** 本仓库并从 `main` 创建分支
2. **一个 PR 只做一件事** — 一个 PR 专注于一个问题。如果你发现有几个不同的问题可以改进，请为每个问题单独开一个 PR
3. **写出有用的 PR 描述** — 说明 *改了什么* 和 *为什么改* ，而不仅仅是差异摘要。如果你的变更是修复 Bug，请提供复现步骤
4. **本地测试** — 提交前请先在本地运行修改后的脚本并验证输出
5. **不要夸大** — 如果你的 PR 描述里声称“包含了测试或行为变更”，请确保在 diff 中体现出来

## 审核流程

- 会尽力审核，通常几天内完成。如果一周没有回应，可以在 PR 中留言提醒
- 审核反馈会具体说明：要改什么、是否为阻塞性问题。如果一个 PR 需要超过约 2 轮才能收敛，可能会被关闭并附上说明——待方向更清晰后可以重新打开
- 针对性修复一般可以直接合并；范围较大的改动通常会 squash-merge 以保持提交历史整洁

## SVG 指南

如果你的贡献涉及 SVG 文件，请遵循 [CLAUDE.md](./CLAUDE.md) 中的技术约束：

- 不要使用：`mask`、`<style>`、`class`、外部 CSS、`<foreignObject>`、`<animate*>`、`<script>`、`<symbol>+<use>`
- 使用 `fill-opacity` / `stroke-opacity` 代替 `rgba()`
- `marker-start` / `marker-end` 有条件允许 — 见 `shared-standards.md` §1.1（必须位于 `<defs>` 中，`orient="auto"`，形状必须为三角形/菱形/椭圆形）
- `<image>` 上的 `clipPath` 有条件允许 — 见 `shared-standards.md` §1.2（必须位于 `<defs>` 中，单个形状子元素，仅用于 `<image>` 元素）
- 所有 SVG 必须使用目标画布格式的正确 `viewBox`

## Bug 报告

在 [GitHub Issues](https://github.com/hugohe3/ppt-master/issues) 上提交 issue，并包含以下内容：

- 问题的清晰描述
- 复现步骤
- 预期行为与实际行为
- 环境详情（操作系统、Python 版本、使用的 AI 编辑器）
