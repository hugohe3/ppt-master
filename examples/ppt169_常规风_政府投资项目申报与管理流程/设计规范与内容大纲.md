- **项目名称**：政府投资项目申报与管理流程讲解

## 一、设计规范

### 1.1 配色方案

采用专业咨询风格配色，强调权威性与可信度：

**主色调：McKinsey 蓝色系**

- 主蓝色：`#005587`（标题、关键信息）
- 深蓝色：`#003F6C`（强调元素）
- 浅蓝色：`#4A90E2`（辅助图形、连接线）

**辅助色**

- 深灰色：`#333333`（正文文字） -中灰色：`#666666`（次要信息）
- 浅灰色：`#E8E8E8`（背景、分隔线）

**功能色**

- 警示红：`#D9534F`（风险提示、必备条件）
- 强调橙：`#F0AD4E`（关键节点）
- 成功绿：`#5CB85C`（完成状态）

**背景色**

- 主背景：`#FFFFFF`（纯白）
- 卡片背景：`#F8F9FA`（浅灰白）

### 1.2 字体系统

采用系统 UI 字体栈，确保跨平台一致性：

```
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "Microsoft YaHei", "PingFang SC", sans-serif
```

**字号规范（16:9 - 1280×720）**

- 封面主标题：60px，font-weight: 700
- 封面副标题：28px，font-weight: 400
- 页面标题：42px，font-weight: 700
- 章节编号：36px，font-weight: 700
- 一级内容：26px，font-weight: 600
- 二级内容：22px，font-weight: 400
- 正文内容：20px，font-weight: 400
- 注释说明：16px，font-weight: 400

### 1.3 布局基线

**页面边距**

- 左右边距：60px
- 上边距：50px（标题区）
- 下边距：40px（页码区）

**标题区（统一）**

- 位置：y=50px
- 高度：60px
- 标题文字：x=60px, y=85px
- 章节编号与标题间距：20px

**内容区**

- 起始位置：y=130px
- 可用高度：540px（130px 至 670px）
- 左右内容区：60px 至 1220px（可用宽度 1160px）

**流程图规范**

- 节点宽度：180-220px
- 节点高度：70-90px
- 节点间距：60-80px（横向），50-70px（纵向）
- 箭头线宽：3px
- 圆角半径：8px

**表格规范**

- 表头高度：50px
- 行高：45px
- 单元格内边距：12px
- 边框颜色：`#D0D0D0`
- 边框宽度：1px

### 1.4 视觉元素规范

**分隔线**

- 粗分隔线：高度 4px，颜色 `#005587`（用于章节过渡）
- 细分隔线：高度 1px，颜色 `#E8E8E8`（用于内容区分）

**图标系统**

- 尺寸：32×32px 或 40×40px
- 风格：线性图标，线宽 2px
- 颜色：与主色调保持一致

**强调框**

- 边框：2px solid `#D9534F`（警示）或 `#F0AD4E`（提示）
- 圆角：6px
- 内边距：20px
- 背景：`#FFF9F5`（浅橙）或 `#FFF5F5`（浅红）

### 1.5 信息可视化原则

遵循高端咨询风格的核心原则：

1. **清晰的层级结构**：通过颜色、字号、粗细建立明确的信息层级
2. **充足的留白**：避免信息拥挤，保持视觉呼吸感
3. **一致的对齐**：所有元素严格遵循网格对齐
4. **精简的文字**：每页核心信息不超过 3-5 个要点
5. **专业的图表**：流程图、对比表、矩阵图为主要可视化手段

### 第 20 页：第七部分 - 总结

**页面标题**：五大核心战略建议

**实操建议**

1. **采纳"双轨并行"策略**
   - A 轨：技术审批（在线平台）
   - B 轨：财务预算（项目库+年度计划）
2. **彻底掌握"在线平台"**
   - 唯一法律入口
   - 精准选择管辖层级
3. **前置启动"关键要件"**
   - 用地预审、节能审查等
   - 不要等可研才申请
4. **适应"成都决策后移"新规**
   - 可研阶段是最关键决策点
   - 投入最优资源编制
5. **早期厘清"融资模式"**
   - 经营性项目引入国资审批
   - 提前规划双重审查

**可视化**

- 每个要点配简洁图标

## 三、执行者注意事项

### 3.1 SVG 生成规则

- 画布：viewBox="0 0 1280 720"，width="1280"，height="720"
- 背景：使用 `<rect>` 元素，填充 `#FFFFFF`
- 禁止使用 `<foreignObject>`，文本换行使用 `<tspan>`
- 所有元素严格对齐网格

### 3.2 文件命名规范

- 格式：`slide_XX_description.svg`
- 示例：
  - `slide_01_cover.svg`
  - `slide_02_contents.svg`
  - `slide_03_approval_hierarchy.svg`

### 3.3 流程图绘制要点

- 使用 `<rect>` 绘制矩形节点（圆角 8px）
- 使用 `<path>` 或 `<polyline>` 绘制箭头
- 箭头末端使用 `<marker>` 定义箭头头部
- 节点填充色：`#F8F9FA`，边框：`#005587`，线宽：2px

### 3.4 表格绘制要点

- 使用 `<rect>` 和 `<line>` 绘制表格框架
- 表头背景：`#005587`，文字：`#FFFFFF`
- 偶数行背景：`#F8F9FA`，奇数行：`#FFFFFF`
- 文字垂直居中：使用 `dominant-baseline="middle"`

### 3.5 质量检查点

- ✓ 所有文字可读性（字号、颜色对比度）
- ✓ 对齐精度（使用标尺检查）
- ✓ 颜色一致性（严格使用规范色值）
- ✓ 留白充足（避免信息拥挤）
- ✓ 层级清晰（通过字号、粗细、颜色区分）
