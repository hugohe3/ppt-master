### 第 4 页：三种对象类型详解

- **标题**: Git 的三种核心对象
- **内容**:
  - **Blob**: 存储文件内容（紫色）
    - 只包含原始数据
    - 无文件名和路径
  - **Tree**: 表示目录结构（绿色）
    - 指向 Blobs 和子 Trees
    - 记录文件名、权限、哈希
  - **Commit**: 代表快照（黄色）
    - 指向顶层 Tree
    - 包含作者、消息、父提交
- **布局**: 三列卡片布局

### 第 5 页：对象链接关系

- **标题**: 对象如何串联起历史
- **内容**:
  - 流程图展示：Commit → Tree → Blobs/Trees
  - 编号说明：
    1. Commit 指向顶层 Tree
    2. Tree 指向 Blobs 和子 Trees
    3. Blob 保存文件内容
    4. 单个哈希确定完整状态
- **布局**: 中央流程图 + 步骤说明

### 第 6 页：底层探索命令

## 设计原则

### CRAP 原则应用

1. **对比 (Contrast)**: 使用色彩编码区分三种对象类型
2. **重复 (Repetition)**: 保持卡片样式、图标风格一致
3. **对齐 (Alignment)**: 所有元素严格网格对齐
4. **亲密性 (Proximity)**: 相关内容分组，使用空白分隔

### 技术内容呈现

- **代码可读性**: 深色背景 + 等宽字体 + 充足行距
- **概念可视化**: 使用图标、流程图、色彩编码
- **层次清晰**: 主次分明的标题层级
- **留白充足**: 避免信息过载
