### 🤖 What are Claude Agent Skills?

The video introduces `Claude Skills` (or `Agent Skills`) as a game-changing feature [[00:06](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=6)]. The easiest way to understand them is as a `reusable instructions menu` [[00:38](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=38)]. These Skills teach Claude how you want a task done, what tools to use, and what standards to follow, allowing it to execute tasks automatically when needed [[00:45](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=45)].

Before Skills, users had to repeat the same complex instructions in every new chat to, for example, maintain a specific `brand voice` [[00:54](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=54)]. While `Custom Instructions` in `Projects` were a partial solution [[01:08](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=68)], they had two fundamental limitations:

1. **Not Portable**: The instructions were locked inside that specific project [[01:16](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=76)].
2. **Not Stackable**: A `copywriting` project and a `campaign analysis` project operated in "complete isolation" and couldn't be combined for a new task [[01:32](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=92)].

`Claude Skills` solve this by extracting workflow instructions and packaging them at the `account level`, making them available across the web app, API, and code [[01:56](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=116)].

### 🔑 The Claude Ecosystem: Skills vs. MCP vs. Projects

The video clarifies a common misconception, explaining that Skills, MCP, and Projects are not competing but are `building blocks` designed to coexist [[03:31](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=211)]:

- **`Claude`**: The core `brain` or intelligence that makes decisions [[02:48](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=168)].
- **`Skills`**: The `instruction menu` defining *how* specific tasks should be done [[02:57](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=177)].
- **`MCP` (Multi-Component Platform)**: Provides the `tools` for capabilities outside of Claude's knowledge, like calling a database or creating a file [[02:57](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=177)].
- **`Projects`**: The `dedicated workspace` where you combine Skills, MCP tools, and background knowledge for ongoing work [[03:05](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=185)].

### ✨ The 3 Types of Skills

The video outlines three main types of skills available to users:

1. **`Official Skills`**: Built by Anthropic. These are ready-to-use skills found in `Settings > Capabilities` [[03:45](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=225)]. The video recommends only turning on the ones you will actually use, as activating too many can confuse Claude [[04:05](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=245)]. There are also `Partner Skills` from companies like `Notion` [[04:19](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=259)].
2. **`Custom Skills`**: Your own creations [[04:34](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=274)]. You can create these by uploading a skill file or, if non-technical, by using the official `Skill Creator` skill [[04:52](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=292)].
3. **`Community Creator Skills`**: Skills created by other users, which should be used with `cautious about security` [[05:04](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=304)].

**Pro-Tip**: The video notes that Skills might not always trigger automatically. The recommended tip is to add a line to your `account-level custom instructions` to "always consider relevant skills when responding" [[05:24](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=324)].

### 🛠️ How to Create & Use Custom Skills

The video demonstrates three primary methods for creating custom skills:

### 1️⃣ Method 1: Extend an Existing Official Skill

This is the easiest way to start. The video shows an example of creating a `Branded Deck` skill [[08:44](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=524)].

- **Action**: Upload your company's `branded presentation template` [[08:30](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=510)].
- **Prompt**: Ask Claude to read the `official PowerPoint building skills`, study the uploaded template (to extract colors/fonts), and "create a new skill" that extends the official one [[08:36](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=516)].
- **Result**: Claude generates a `zip file` for the new custom skill. You then upload this file in your settings [[09:24](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=564)]. When you later ask for a "company branded presentation deck," Claude will trigger your new skill [[10:05](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=605)].

### 2️⃣ Method 2: Package an Existing Workflow into a Skill

This is an immediate quick win for porting your existing work [[11:56](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=716)].

- **Action**: Take an existing `Project` that has detailed custom instructions (e.g., a `team reporting` project that pulls data from a `Notion` database) [[12:03](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=723)].
- **Prompt**: Turn on the `Notion MCP` [[12:31](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=751)]. Then, paste your *entire* set of custom instructions and workflow details into a prompt, asking Claude to package it as a new skill (e.g., `Notion Project Dashboard` skill) [[12:38](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=758)].
- **Result**: You can now generate that specific dashboard `in any new chat` [[13:20](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=800)], as the workflow is no longer locked in the original project.

### 3️⃣ Method 3: Build Multiple Skills from Scratch

This is the most advanced method, shown for a blog content creation workflow [[14:01](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=841)].

- **Action**: Create two separate skills from scratch:
    1. A `Keyword Research` skill that is instructed to use `Ahrefs MCP` tools [[14:08](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=848)].
    2. A `Blog Writer` skill that includes output guidelines for `AI search engines` [[14:43](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=883)].
- **Usage**: In a new `Project`, you add these skills and instruct the project's custom instructions to use `Keyword Research` and `Blog Writer` for specific tasks [[15:13](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=913)].
- **Result**: A simple prompt like "create a new blog post about..." triggers *both* skills, calls the `Ahrefs MCP` for keywords, and generates a fully formatted Word document following the complex workflow [[15:45](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=945)].

### 💡 Final Advice & Guidelines

- **When to create a skill?** Ask yourself three questions [[16:40](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=1000)]:
    1. Do you repeat the same instructions 3+ times?
    2. Would you need to train a real human on this process?
    3. Is quality/format consistency needed every time?
    - If you check "yes" to two or more, `you should create a skill` [[17:01](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=1021)].
- **`Token` Usage**: Be mindful of your token limit on the Pro plan. Only create skills for your `most repeated workflows` [[17:07](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=1027)]. You can even use `ChatGPT` to study the official `Skill Creator` zip file and generate new skills for you to save on Claude tokens [[17:20](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=1040)].
- **Maintenance**: Skills are static. If an `MCP tool` they call changes, the skill might break. The video suggests asking Claude to include a `backup plan` in the skill's instructions in case a tool call fails [[17:33](http://www.youtube.com/watch?v=m-5DjcgFmfQ&t=1053)].
