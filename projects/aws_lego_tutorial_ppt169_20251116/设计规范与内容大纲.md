## 设计实施要点

### CRAP 原则应用

1. **对比 (Contrast)**：
   - 使用 AWS Orange (#FF9900) 突出关键服务
   - 问题用红色边框，解决方案用绿色边框
   - 标题字号大（48px）vs 正文小（20px）

2. **重复 (Repetition)**：
   - 每页左上角保留一致的标题区域
   - 服务图标风格统一（emoji + 圆形背景）
   - 卡片阴影、圆角统一规格

3. **对齐 (Alignment)**：
   - 所有元素基于 60px 左边距网格
   - 卡片内容左对齐
   - 标题区域固定高度 120px

4. **亲密性 (Proximity)**：
   - 相关服务（如 EC2+VPC）放在同一区域
   - 问题-解决方案成对出现
   - 用空白分隔不同概念

### 技术约束

- ✅ 使用 `<tspan>` 手动换行（禁止 `<foreignObject>`）
- ✅ 背景使用 `<rect>` 元素
- ✅ 所有文本使用系统 UI 字体栈
- ✅ 图标优先使用 emoji + SVG 几何形状
- ✅ 架构图使用 `<path>` 绘制连接线和箭头
